# 线下模式视觉小说风格美化计划

## 目标
将线下模式聊天界面重新设计为**视觉小说/漫画风格**，保持所有功能不变，仅做视觉美化。

---

## 一、当前问题分析

### 1.1 现有布局
- 每条消息都有独立头像和名称
- 用户/AI 消息分别显示在左右两侧
- 段落间距过大，不够紧凑

### 1.2 渲染逻辑
- `renderContent()` 函数将内容按 `‖` `（）` `【】` `「」` 标签分割
- 但渲染时没有区分对齐方式

---

## 二、计划任务

### 任务 1：重构消息渲染逻辑 (renderContent)
- **目的**：根据内容类型自动应用对齐样式
- **修改**：
  - 旁白 `‖‖`：居中对齐
  - 动作 `（）`：居中对齐
  - 场景信息 `【】`：居中对齐
  - 对话 `「」` 或普通文本：根据 role 左/右对齐
- **保留**：所有现有功能（心声、标签过滤等）

### 任务 2：视觉小说布局重构
- **核心改变**：
  - 每轮对话（用户+AI）作为一个视觉整体
  - 每轮只显示一次头像（在内容开始处）
  - 头像与内容分离，形成漫画分镜感
- **实现方式**：
  - 修改 template 中的消息渲染结构
  - 使用 CSS 实现漫画分格效果

### 任务 3：段落间距优化
- **目标**：紧凑的段落间距
- **修改**：
  - 消息内段落间距：4px
  - 消息块之间间距：12px
  - 对话与动作/旁白之间有明显区分但不过大

### 任务 4：视觉小说样式美化
- **添加样式**：
  - 漫画分格边框效果
  - 角色名标签（类似漫画对话框名字）
  - 对话气泡效果（可选）
  - 阴影和光影增强层次感
- **保留样式**：
  - 旁白：衬线字体、电影字幕风格
  - 动作：灰色斜体
  - 对话：白色高对比度

### 任务 5：功能验证
- 验证所有交互功能正常：
  - 长按/右键菜单
  - 心声按钮显示
  - 头像动画
  - 输入状态指示器

---

## 三、预期效果

### 3.1 布局示意
```
┌─────────────────────────────┐
│  [头像] 角色名              │
│  ┌─────────────────────┐   │
│  │ 对话内容...         │ → 右对齐
│  │（动作描写）         │ → 居中
│  │‖旁白描写‖           │ → 居中
│  │【场景信息】         │ → 居中
│  └─────────────────────┘   │
│                             │
│  [头像] 用户名              │
│  ┌─────────────────────┐   │
│  │ 对话内容...         │ ← 左对齐
│  │（动作描写）         │ → 居中
│  └─────────────────────┘   │
└─────────────────────────────┘
```

### 3.2 视觉风格
- 漫画分格感
- 紧凑但清晰的段落
- 居中对齐的动作/旁白
- 左右对齐的对话

---

## 四、注意事项

1. **不改变**：底层交互逻辑（长按、滑动、心声等）
2. **保持兼容**：所有现有标签和格式仍然有效
3. **渐进增强**：先实现基础功能，再美化视觉
