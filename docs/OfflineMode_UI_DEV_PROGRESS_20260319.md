# 线下模式 UI 开发记录
更新时间：2026-03-19

这份文档用于记录线下模式当前的视觉方向、已落地改动、核心入口和后续待办，避免中断后丢失上下文。

## 目标
线下模式不再沿用“微信聊天流”视觉，而是改成更接近“轻视觉小说 / 叙事阅读页”的界面。

核心要求：
- 保留现有功能和组件能力，不删功能。
- 手机端消息菜单改成长按触发，桌面端改为右键触发。
- 文本消息按叙事类型拆分渲染，而不是统一气泡。
- HTML 卡片、图片、红包、转账等特殊消息继续正常渲染，但外观要弱化“聊天卡片感”。
- 整体界面参考浅灰白、安静、留白多的阅读页。

## 当前视觉方向
参考图的目标风格已经明确：
- 顶部：大标题 + 日期/时间/状态元信息。
- 中部：窄栏阅读流，旁白和动作用白色圆角卡片承载。
- 对话：无气泡，头像 + 名字 + 正文排版。
- 底部：轻量工具栏 + 大白色输入胶囊。
- 整体：浅灰底、背景图只做很淡的氛围层，不抢正文。

## 代码入口

文件：
`src/views/WeChat/OfflineModeChatWindow.vue`

职责：
- 线下模式整体布局。
- 顶部标题区、消息滚动区、底部输入区。
- 线下消息过滤。
- 长按 / 右键菜单。
- 心声、背景、行动面板、红包、转账等弹层入口。

本轮已改：
- 顶部改成浅色叙事头部。
- 加入 `sceneDisplayTitle`、`currentDateLabel`、`sceneStatusLabel`。
- 背景改成低透明度 + 白色蒙层，不再是暗色玻璃风。
- 消息区宽度调成阅读列。
- 非文本特殊消息增加浅色“剧情插入物”外壳。
- 输入区外层改成浅色渐变收口。

### 2. 线下文本渲染器

`src/views/WeChat/components/TheaterMessageRenderer.vue`

- 将线下文本拆成不同叙事段落。

本轮已改：

- 旁白/动作改成白卡片。
- 对话字号下调，更接近手机阅读感。

本轮已改：

- 输入框改成白色大胶囊。
- 发送按钮保留，但整体收敛。

### 4. 消息解析规则
文件：
`src/utils/chatMessageDisplay.js`

职责：
- 统一线上/线下可见性判断。
- 解析 `[ONLINE] / [OFFLINE]`。
- 解析 `INNER_VOICE`。
- 解析场景 / 旁白 / 动作 / 对话段落。

当前依赖的方法：
- `shouldShowInOfflineMode`
- `isOfflineTextMessage`
- `parseOfflineSegments`
- `extractInnerVoiceData`
- `extractLatestOfflineScene`

## 当前交互规则
- 手机端消息菜单：`450ms` 长按触发。
- 触摸移动超过 `10px`：取消长按菜单。
- 桌面端消息菜单：右键触发。
- 单击消息：不应触发菜单。

## 已完成的关键点
- 线下文本消息不再走普通聊天气泡，而是走专用叙事渲染器。
- 只有普通对话会参与头像显示逻辑。
- NPC tagged 对话不显示头像。
- 旁白、动作、场景不显示头像。
- HTML / 图片 / 红包 / 转账等特殊消息保留原有功能，不再误进剧场文本渲染。
- 线下页面主结构已切到浅色阅读页方向。

## 仍需继续优化

### 1. 顶部还可以更像参考图
还可以继续收：
- 标题字号和留白再细调。
- 元信息图标和字重再压轻一点。
- 按钮边框和阴影还可以更淡。

### 2. 特殊消息外观还不够统一
目前只是给了浅色外壳，后续还要继续分化：
- HTML 卡片
- 图片
- 红包 / 转账
- 语音 / 文件

目标是看起来像“剧情插入物”，而不是聊天卡片。

### 3. 消息密度还要继续真机微调
目前已经把字号压了一轮，但还需要继续看：
- 对话行高是否偏松。
- 旁白卡片高度是否偏高。
- 场景标签是否还太显眼。

### 4. 仓库里仍有历史乱码
本轮清掉了线下模式相关的主要显示乱码，但 `ChatWindow.vue` 这类历史文件仍可能有残留脏字符串。
如果后续构建再次报 `500` 或模板错误，优先排查这些旧文件。

## 回归检查清单
- 线下模式顶部标题、日期、状态显示正常。
- 文本消息按场景 / 旁白 / 动作 / 对话正确分层。
- 连续对话头像只出现一次。
- 用户对话在右侧，角色对话在左侧。
- NPC tagged 对话不带头像。
- HTML 卡片能正常解析，不显示 JSON 原文。
- 手机端轻点消息不弹菜单，长按才弹。
- 桌面端右键菜单正常。
- `npm run build` 可以通过。
